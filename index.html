ly-input 封装过程

今天开始了minily-ui 中 input 输入框的组件封装，继续昨天的开发与完善

  • 先上一张图看下 LyBotton 组件完善的情况：
  • 可以看到，虽然比较简单，也比较基础，但好歹也是完成了一部分，后续进一步完善和增加 新特性 ，会越来越丰富的
1. 话不多说，新建 input 组件所需的文件夹以及对应的文件

  • 1-1.如下图：

    • 图中三个文件和之前 button 提及到的三个文件是起着一样的作用的，这里不再做过多的叙述

  • 1-2.看一下 LyInput 组件内部都暴露了哪些公共属性以及方法。如下图

    • 可以看到，在 input 外层增加了一层 div 标签，并且继承着两个方法，@mouseenter，@mouseleave
      这两个方法分别监听这鼠标 移入 和 移出 事件，分别设置 hovering 的不同状态。

    • input 上面则暴露出更多的属性以及方法，:vulue 用来接收输入的内容，:placeholder 用来设置默认状态下的输入框文本提示语，:disabled 判断是否禁止，等等

    • 我为 input 预留了 许多空间，为此它的自定义性将更高，以下列举几种常见的情形，如图

      • 引用的组件通过传入不同的属性，即可使用不用的功能

      • clearable 属性可使用清除功能，可清空输入框内容，showPassword 属性可让输入框变为密码框，其内容可见与否的功能。size 有 mini，small， medium，large 四种，分别对应四种不同大小的输入框。

      • 其他属性不再进行一一介绍，用法大同小异

  • 1-3.编辑 组件目录下的 index.js 文件. 和前一天说明的是一样的，没有什么区别

  • 1-4.编辑 package 目录下的 index.js 文件. 将新增的 LyInput 组件加进去，用蓝线圈出的地方是关键，这样做确保了 install 方法中 components.forEach 之后 能够注册全部的组件，之后所有的新增组件只需要在这里引进就好了

  • 问题：npm publish 可能会遇到失败的情况，请记得去修改 package.json 文件中的 version 字段的值，同一个版本不可重复提交

MiniLY-UI 开发记录

开发项目时，我们引用过许多的第三方ui框架，却少有自己动手写过一款属于自己的ui框架。此处记录的时minily-ui从零开始的过程

快速开始

1.创建一个vue项目
  • ui框架名称，建议先去 npm官网 进行搜索，已经被提交使用的名称无法再次使用，后面上传到npm会用到这个ui名称的
2.创建第一个组件目录以及相对应的文件

  • package文件夹用来存放组件，每个组件文件夹中包含3个文件，index.js, index.vue, index.scss

  • 编写第一个组件，这里以 button 为例，也就是上述的 index.vue 文件

  • 然后在对组件文件夹中相对应的 index.js 进行如下配置，其中值得注意的是：LyButton.name，此处的 name 是index.vue 中 export default 中的 name 属性，两者相对应

  • index.scss 文件这里不做过多说明，它用来设计你的组件样式，让你的组件看起来更美

3.在 package 目录下创建 index.js 文件，用来注册所有的组件

  • 这里的 index.js 文件，用 vue 的 install 方法将所有的组件全部注册

  • 到了这里，基本的内容就完成了，此时便可以在 App.vue 中 import 这个 LyButton 组件使用了。如下图：

  • 问题，如何把这个ui，放到其他的项目中使用，像其他的框架那般，npm install 安装即可？

  • 1、对webpack.config.js 进行配置，修改 entry，output 两处配置

  • 2、对package.json 进行配置

  • 关键，这里的路径必须和webpack.config.js中的filename一致,否则你在项目引入的时候会报错

  • 如果不设置的话，你在项目引入你发布的包的时候需要import包的相对路径，如果层级过多，在node_modules里面一个个找，很麻烦

4.最后，可以打包测试了，不出意外的话

  • 1、打包

  • 经过以上两步，项目目录中会生成 minily-ui-1.0.0.tgz 的一个文件

  • 2、新建一个测试项目, 并将 minily-ui-1.0.0.tgz 文件放入新建的项目根目录中并安装

  • 3、在 main.js 中引入 minily-ui

  • 4、可以在任意一个vue页面使用你的 ui 组件了，此处以 App.vue 为例

  • 结果：可以看到，此时通过安装 minily-ui-1.0.0.tgz ，是可以正常使用我们的ui组件的，下面把它提交到 npm 网站上

  • 5、npm 发布你的 UI 组件

    • 5-1，去 npm官网 注册一个账号
    • 5-2，登录 npm 并推送自己的代码
    • 5-3，如果你在发布的时候一切都正常，那么恭喜你，很幸运。现在可以通过以下指令安装了
    • 5-4，可能会遇到的问题，提示需要验证邮箱，你只需到你的邮箱里确认就好
    • 5-5，npm publish 之前记得先 npm build ，否则 npm 中的 dist 目录下的 minily-ui.js 是没有更新的
  • 有关于MiniLY-UI的更多信息: 前往npm官网
